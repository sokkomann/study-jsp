전제 -> 서블릿은 JAVA, JSP는 HTML
자바 파일에 extends HttpServelet (상속받으면 서블릿이됨)
web.xml은 이정표
JSP는 조회따위, 서블릿은 처리따위를
get은 단순페이지 이동

JSP(Java Server Page)
	HTML을 중심으로 자바와 연동하여 사용하는 웹 언어이다.
	HTML 코드 안에 JAVA 코드를 작성할 수 있는 언어이다. = HTML
	(JAVA 안에 HTML은 서블릿) = JAVA
	JSP는 조회따위, 서블릿은 처리따위를

서버
	사용자의 요청에 맞는 서비스를 제공해주는 것

	- 요청(Request)	: 클라이언트 --> 서버
	- 응답(Response)	: 서버 --> 클라이언트

웹 서버 - Apache
	사용자의 요청이 정적 데이터인지 동적 데이터인지 판단한다.
	정적이면 미리 준비한 HTML 문서를 그대로 응답,
	동적이면 서버-와스-웹 컨테이너 경로로 요청을 보냄(정적으로 처리해)

웹 컨테이너(서블릿 컨테이너 라고도 부르자)
	동적 데이터일 경우 JSP 또는 서블릿으로 연산 및 제어 그리고 DB까지 접근한다.
	DB에 접근하는 연산을 복잡한 연산이라고 함. -> JAVA로 처리한다.
	동적 데이터가 정제되면(정적 데이터로) 이를 웹 서버를 통해 응답해준다.

WAS(Web Application Server) - Tomcat(우리는 톰캣)
	동적 데이터를 처리할 서블릿을 메모리에 할당하며(객체화 해준다는 얘기), 
	web.xml을 참조한 뒤 알맞은 서블릿에 대한 Thread를 생성한다. 
	서블릿 요청과 응답 객체 생성 후에 서블릿에 전달하면,
	연산을 종료하고 메모리에서 해제시킨다.

서블릿(Servlet)
	JAVA 코드 안에 HTML 코드를 작성할 수 있는 JAVA 프로그램이다.
	Thread에 의해 서블릿에 있는 service() 메소드가 호출된다.
	전송 방식에 따라 doGet() 혹은 doPost() 등의 메소드가 호출된다.

요청 방식(자바에 보낼때 무슨 요청방식을 사용하지?)
	- GET	
		주소(자바주소말고 웹)에 데이터를 추가하여 전달하는 방식
		보통 쿼리 스트링(Query string)(디비쿼리말고)에 포함되어 전송되므로, 길이에 제한이 있고,
		주소에 데이터가 보이므로 보안상 취약점이 존재한다.	
		ex)https:www.naver.com?news=코리아신문
		에서 물음표가 있네? -> GET방식
		=> 중요한 데이터나 길이가 긴 데이터면 POST방식으로 요청
		근데 GET방식이 POST방식보다 상대적으로 빠른 전송방식임.

	- POST
		데이터를 별도로 첨부(Header에 첨부)(화면 헤더말고 담기는 객체로 생각)
		하여 전달하는 방식.
		브라우저 히스토리에도 남지 않고 데이터는 쿼리 스트링과는 별도로 전송됨.
		따라서 데이터의 길이에 제한도 없고, GET 방식보다 보안성이 높음.
		근데 GET방식보다 상대적으로 느림
	
응답 방식
	- forward
		요청한 경로가 그대로 남아있기 때문에 응답 페이지를 보더라도
      		요청한 경로가 표시된다.
     		.jsp로 이동해야 할 때에는 확장자가 표시되지 않게 하기 위해서
      		forward 방식을 사용한다.
      		단순 페이지 이동 혹은 응답 페이지에 필요한 데이터를 
      		전달하고자 할 때 사용하는 방식이다.
	- redirect
		요청한 경로는 초기화되기 때문에 응답한 경로가 표시된다.
      		다른 서블릿으로 요청해야 할 때에는 이전 요청 경로가 표시되면
      		안되기 때문에 redirect 방식을 사용한다.
      		DB 연산 등 복잡한 연산 수행 후 응답 페이지로 이동해야 할 때,
      		응답 페이지로 이동시켜주는 다른 서블릿으로 요청할 경우 사용하는 방식이다.

forward 때 getContextPath() 를 붙이면 오류남
forward는 기본적으로 경로가 a-intro/ (내 워크스페이스기준)로 시작하기 때문에 
getContextPath() 를 붙이면 a-intro/a-intro 로 되기때문에 오류
